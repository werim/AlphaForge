# AlphaForge Patch Skill

## Workflow

1. Read AGENTS.md
2. Read VERSION.md
3. Read REPORT.md
4. Inspect relevant files
5. Explain current behavior
6. Propose minimal patch
7. Apply patch
8. Run tests
9. Update:
   - VERSION.md
   - REPORT.md
   - CHANGELOG.md
10. Summarize risks

## Constraints

- no architecture rewrite
- preserve lifecycle integrity
- preserve BACKTEST/PAPER/LIVE alignment
- no duplicate alphaforge packages
